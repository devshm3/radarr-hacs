# Screenshots

Drop PNG (or JPG) captures here using the exact filenames below — the main
`README.md` references them. Until a file exists, its spot in the README shows a
broken-image icon, so add them before publishing the release.

| Filename | What to capture |
|----------|-----------------|
| `hero.png` | The card on a dashboard — your best-looking single shot (used at the top of the README) |
| `glass-dark.png` | Appearance: **Glass**, HA in **dark** mode |
| `glass-light.png` | Appearance: **Glass**, HA in **light** mode |
| `material-dark.png` | Appearance: **Material You**, HA in **dark** mode |
| `material-light.png` | Appearance: **Material You**, HA in **light** mode |
| `detail-panel.png` | A poster clicked open, showing the inline detail panel |
| `add-movie.png` | Add mode with a TMDB search result |

Tips:
- Capture at a consistent card width so the gallery rows line up.
- Use the dev instance themes (`glass_test` / `solid_test`, with the Light/Dark
  switch) to produce matching light/dark pairs.
- Keep files reasonably small (PNG, a few hundred KB) so the repo stays light.
